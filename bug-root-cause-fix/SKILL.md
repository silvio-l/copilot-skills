# Bug Root Cause & Fix Skill

## Ziel
Finde die tatsächliche Ursache eines Bugs und implementiere einen kleinen, robusten, architekturtreuen Fix.

## Vorgehen
1. Problemverständnis:
   - erwartetes Verhalten
   - tatsächliches Verhalten
   - Reproduktionsbedingungen
2. Relevante Codebereiche eingrenzen
3. Mehrere Hypothesen bilden und am Code prüfen
4. Wahrscheinlichste Root Cause benennen
5. Mindestens 3 Lösungsoptionen entwickeln:
   - pragmatisch/minimal
   - robust/sauber
   - defensiv/zusätzlich abgesichert
6. Beste Lösung auswählen und begründen
7. Klein und gezielt umsetzen
8. Validieren und Restrisiken benennen

## Regeln
- Keine Symptomfixes ohne Root-Cause-Begründung
- Keine unnötigen Refactorings
- Keine Vermutungen als Fakten
- Änderungen nur im nötigen Umfang

## Ausgabeformat
1. Problemverständnis
2. Relevante Codebereiche
3. Hypothesen
4. Wahrscheinlichste Root Cause
5. Lösungsalternativen
6. Empfohlene Lösung
7. Implementierungsplan
8. Konkrete Änderungen
9. Validierung / Tests
10. Restrisiken
