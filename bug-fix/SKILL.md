# Bug Root Cause & Fix Skill

## Goal
Find the actual root cause of a bug and implement a small, robust, architecture-respecting fix.

## Workflow
1. Problem comprehension:
   - expected behavior
   - actual behavior
   - reproduction conditions
2. Narrow down relevant code areas
3. Form multiple hypotheses and verify against code
4. Name the most probable root cause
5. Develop at least 3 solution options:
   - pragmatic / minimal
   - robust / clean
   - defensive / additionally hardened
6. Select and justify the best solution
7. Implement small and targeted
8. Validate and name residual risks

## Rules
- No symptom fixes without root cause justification
- No unnecessary refactoring
- No guesses presented as facts
- Changes only to the necessary extent

## Output Format
1. Problem understanding
2. Relevant code areas
3. Hypotheses
4. Most probable root cause
5. Solution alternatives
6. Recommended solution
7. Implementation plan
8. Concrete changes
9. Validation / tests
10. Residual risks
